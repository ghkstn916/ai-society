### 피드백 3 — UI 인터랙션 개선 요청

**요청 사항 4가지와 구현 결과**:

#### ① Lesson1_1 — 시대 카드 플립 기능
- **요청**: 옛날 사회 설명이 있는 카드를 누르면 현재 디지털 사회로 변화하는 형식
- **구현**: 농경사회 → 스마트 농업 / 산업사회 → 스마트 팩토리 / 디지털 사회 → 미래 사회 로 플립되는 카드. 다시 누르면 원래로 복귀.

#### ② Lesson1_2 — 키오스크 카드 순서 섞기
- **요청**: 편리한 점과 디지털 격차 문제 순서를 섞어달라
- **구현**: A→A→A→B→B→B 순서에서 A→B→A→B→B→A 순서로 변경

#### ③ Lesson2_1 — 개인정보 드래그 앤 드롭 활동
- **요청**: 학생이 드래그 앤 드롭으로 개인정보 / 개인정보 아님을 구분하는 활동
- **구현**: 10개 항목(주민번호·지문·전화번호·이메일·카드번호·얼굴사진 / 날씨예보·뉴스·도서관위치·위키피디아)을 두 존에 드래그. 제출 후 초록(정답)/빨강(오답) 색상 피드백 제공.

#### ④ Lesson2_2 — 정보 분류 카드 순서 섞기
- **요청**: 보호할 정보와 공유할 정보 순서를 섞어달라
- **구현**: A→A→A→A→B→B→B→B 순서에서 A→B→A→B→A→B→A→B 순서로 변경

## 3. 배포 과정 이슈 및 해결

### 이슈 1 — GitHub Pages 빈 화면
- **증상**: 배포 후 사이트에 접속하면 빈 화면만 표시됨
- **원인**: `BrowserRouter`에 `basename` prop이 없어 `/digital-society-2/` 경로를 인식하지 못함
- **해결**: `<BrowserRouter basename="/digital-society-2">` 추가 + `vite.config.js`에 `base: '/digital-society-2/'` 설정

### 이슈 2 — SPA 새로고침 404
- **원인**: GitHub Pages는 정적 호스팅이라 React Router 경로로 직접 접근 시 404 반환
- **해결**: `public/404.html`에 sessionStorage 리다이렉트 스크립트 추가, `index.html`에서 복원 처리

### 피드백 4 — 학생 활동 상태 localStorage 저장

**요청**: 퀴즈 답변, 체크리스트, 드래그앤드롭 배치 등 학생이 체크하고 넘어간 것들이 페이지를 다시 방문해도 초기화되지 않고 유지되게 해달라 (쿠키 삭제나 컴퓨터 끄기 전까지 유지)

**구현 방식**:

- `src/hooks/usePersistentState.js` 커스텀 훅 신규 생성
  - `key`가 있으면 localStorage에 JSON 직렬화하여 저장, 없으면 일반 `useState`처럼 동작
- 4개 인터랙티브 컴포넌트에 `storageKey` prop 추가 (`null`이면 비저장, 하위 호환 유지)

| 컴포넌트 | 저장 상태 |
|---------|---------|
| `ChoiceQuiz` | 선택한 답 |
| `FlipReveal` | 답 공개 여부 |
| `SortCards` | 분류 답변 + 정답 확인 여부 |
| `FormativeQuiz` | 답변 배열 + 제출 여부 |

- 9개 레슨 파일 + 3개 Quiz 파일에 `storageKey` props 전달 및 인라인 활동 상태 교체

| 저장되는 인라인 상태 | 레슨 |
|-------------------|------|
| 플립카드 토글 상태 | 1-1 |
| 디지털/아날로그 판단 선택 + 정답 공개 여부 | 1-1 |
| 디지털 격차 해결방안 선택 | 1-2 |
| 진로 탐색 텍스트 입력 + 제출 여부 | 1-3 |
| 드래그앤드롭 풀/존 배치 + 제출 여부 | 2-1 |
| 비밀번호 체크리스트 | 2-3 |
| CIA 위협 분석 선택 + 정답 공개 여부 | 3-1 |
| 보안 실천 체크리스트 + 시나리오 선택 | 3-2 |
| 상황별 규칙 공개 여부 + 나만의 규칙 텍스트 + 저장 여부 | 3-3 |

- UI 전용 상태(아코디언 open/close, 필터, 직업 카드 선택 패널 등)는 persist 제외

**키 네이밍 컨벤션**: `dc-m{모듈번호}l{레슨번호}-{타입}-{인덱스}` (예: `dc-m1l1-quiz-0`)

### 피드백 5 — 학생 완료 기록 수집 + 어드민 페이지

**요청**: 완료 버튼을 누를 때 학생 이름과 퀴즈 결과를 저장하고, 선생님만 볼 수 있는 어드민 페이지를 만들어달라.

**구현 방식**:

- `src/data/quizAnswers.js` 신규 생성
  - `lessonQuizInfo`: 레슨별 퀴즈 정답 키와 formative 정답 배열 매핑
  - `collectSubmission()`: localStorage에서 퀴즈 선택값을 읽어 정답 여부를 계산하고 제출 객체 반환
  - `saveSubmission()` / `loadSubmissions()` / `deleteSubmission()` / `clearSubmissions()`: `dc-submissions` 키로 localStorage CRUD

- `src/components/layout/LessonNav.jsx` 수정
  - 완료 버튼 클릭 시 학생 이름/학번 입력 모달 표시
  - 이름 확인 후 `collectSubmission` → `saveSubmission` → `markComplete` 순으로 실행
  - 모달 닫기: 취소 버튼 / 배경 클릭 / Escape 키, 확인: Enter 키

- `src/pages/AdminPage.jsx` 신규 생성
  - 비밀번호 게이트 (기본값 `teacher2024`, sessionStorage로 세션 유지)
  - 제출 목록: 모듈 필터 + 학생 이름 검색, 행 클릭 시 퀴즈 정오답 + 형성평가 점수 펼침
  - CSV 내보내기 (BOM 포함, 한글 Excel 호환)
  - 개별 삭제 + 전체 삭제 (확인 다이얼로그)

- `src/main.jsx` 수정: `/admin` 경로 추가

| 항목 | 내용 |
|------|------|
| 어드민 URL | `/digital-society-2/admin` |
| 기본 비밀번호 | `teacher2024` (파일 상단 `ADMIN_PASSWORD` 상수에서 변경 가능) |
| 데이터 저장 위치 | localStorage `dc-submissions` |
| 한계 | 기기별 localStorage에 저장되므로 학생 기기의 데이터를 선생님 기기에서 직접 확인 불가 → CSV로 전송하거나 추후 백엔드 연동 필요 |

### 버그 수정 1 — 형성평가 페이지 LessonNav 누락

**문제**: `LessonPage.jsx`에서 `lesson.isQuiz === true`일 때 `LessonNav` 렌더링을 제외하는 조건이 있어, 형성평가 페이지에는 이전/다음 버튼도 없고 완료 버튼이 학생 이름 모달을 건너뜀. 결과적으로 형성평가 완료 기록이 어드민에 저장되지 않았음.

**수정 내용**:

- `LessonPage.jsx`: `{!lesson.isQuiz && <LessonNav />}` 조건 제거 → 모든 페이지에 동일하게 `LessonNav` 표시
- `Quiz1.jsx` / `Quiz2.jsx` / `Quiz3.jsx`: 중복이던 자체 완료 버튼 및 불필요한 import(`useProgressStore`, `Link`) 제거

## 6. 다음 개선 시 참고사항

  - Firebase 연동 시 변경 범위: `saveSubmission` / `loadSubmissions` 함수 교체 + AdminPage 실시간 리스너 추가만으로 적용 가능
